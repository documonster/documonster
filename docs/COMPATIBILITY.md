## Rendering scope

ExcelTS ships a **zero-dependency deterministic preview renderer** for SVG / PNG / PDF, not an Excel-pixel-perfect compositor. Classic charts are driven by a `ChartScene` intermediate representation shared across SVG, PNG, and PDF; ChartEx charts use dedicated geometry collectors that keep the SVG and vector-PDF paths equivalent by construction. The preview is suitable for thumbnails, email attachments, server-side report generation and quick sanity checks; **it is not a replacement for Excel or LibreOffice** when pixel-identical output matters. For production-grade rendering, round-trip the `.xlsx` through headless LibreOffice (`soffice --convert-to pdf`) — the metadata preservation guarantees in this library make that a byte-safe handoff.
